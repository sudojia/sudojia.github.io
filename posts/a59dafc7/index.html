使用 GitHub Actions 自动签到 SSPANEL 面板

发表于2021-09-27|更新于2021-09-27|干货
|字数总计:1.4k|阅读时长:4分钟

写在前面

SSPANEL 面板如果不知道的话，那还是别知道的好，知道的人就可以往下看吧！

关于 SSPANEL 面板自动签到的项目也有，是 Python 写的，部署于腾讯云函数上，但我看了下，腾讯的东西，我也不太敢用！其次就是不支持多个网站，执行一次之后又得改源码换另一个站，然后又执行一次，我又不好意思提 Issues，所以就自己写了一个，本想用 JAVA 写的，想了下还是用弱语言 —— JavaScript

本文禁止讨论✈️场、本文禁止讨论✈️场、本文禁止讨论✈️场！！！

特点

相比 Python 版本的，虽然都是免费！但是我这个支持多个平台多个用户以及多个消息平台（Telegram、server 酱、Bark、PushPlus、钉钉等）服务推送

前期准备

  • GitHub 账号
  • ✈️场的面板一定是 SSPANEL（废话）

适用人群

  • 老少皆宜
  • 有这个需求的人
  • 断签 7 天或者断签某天就会把你号给删的那种 ✈️

使用

  1. 点击下方项目

  2. 右上角 Fork 该项目

  3. 在仓库的 Settings - Secrets - New repository secret 添加变量，变量详细请看变量说明

  4. 点击仓库中的 Actions，图中所示

  5. 看图

  6. 继续看图

  7. 执行成功，如图，如果报错，不妨可以提个 issues，不要害羞！

变量说明

主变量（必须）

NameValue说明属性
SITE_ACCOUNTS网址账号密码单账号填写规则，e.g：https://paolu.com,aaa@gmail.com:123456
多个填写规则：https://aaa.com,aaa@gmail.com:aaa&https://bbb.com,bbb@gmail.com:bbb&...以此类推
网站与账号密码之间用英文逗号（,）分割，账号与密码之间用英文冒号（:）分割
中文说明：网站,账号:密码，多个：网站,账号:密码&网站,账号:密码		必须

消息推送变量（可选）

Name归属属性说明
PUSH_KEY微信 server 酱推送非必须server 酱的微信通知官方文档，已兼容 Server 酱·Turbo 版
BARK_PUSHBARK 推送非必须IOS 用户下载 BARK 这个 APP，填写内容是 app 提供的设备码
例如：https://api.day.app/123 ，那么此处的设备码就是 123
BARK_SOUNDBARK 推送非必须bark 推送声音设置，例如 choo，具体值请在 bark-推送铃声-查看所有铃声
TG_BOT_TOKENTelegram 推送非必须TG_BOT_TOKEN 和 TG_USER_ID 两者必需
填写自己申请 @BotFather的 Token
如 10xxx4:AAFcqxxxxgER5uw
TG_USER_IDTelegram 推送非必须TG_BOT_TOKEN 和 TG_USER_ID 两者必需
私聊它 @userinfobot 随便发点什么即可获取到自己的 ID
DD_BOT_TOKEN钉钉推送非必须(DD_BOT_TOKEN 和 DD_BOT_SECRET 两者必需)官方文档
只需 https://oapi.dingtalk.com/robot/send?access_token=XXX 等于 = 符号后面的 XXX 即可
DD_BOT_SECRET钉钉推送非必须(DD_BOT_TOKEN 和 DD_BOT_SECRET 两者必需) ，密钥，机器人安全设置页面，加签一栏下面显示的 SEC 开头的 SECXXXXXXXXXX 等字符，注：钉钉机器人安全设置只需勾选加签即可，其他选项不要勾选
QYWX_KEY企业微信机器人推送非必须密钥，企业微信推送 webhook 后面的 key 详见官方说明文档
IGOT_PUSH_KEYiGot 推送非必须iGot 聚合推送，支持多方式推送，确保消息可达。 参考文档
PUSH_PLUS_TOKENpushplus 推送非必须微信扫码登录后一对一推送或一对多推送下面的 token(您的 Token)
官方网站
PUSH_PLUS_USERpushplus 推送非必须一对多推送的 “群组编码”（一对多推送下面 -> 您的群组(如无则新建)->群组编码）
注：(1、需订阅者扫描二维码 2、如果您是创建群组所属人，也需点击“查看二维码”扫描绑定，否则不能接受群组消息推送)
只填 PUSH_PLUS_TOKEN 默认为一对一推送

注意事项

  1. 暂不支持密码带英文逗号（ , ）与英文冒号（ : ）的特殊字符！
  2. 关于消息推送变量
    • 如果是 Telegram 通知，那么需要填两个变量，分别是 TG_BOT_TOKEN 和 TG_USER_ID，两者不可缺一
    • 如果是钉钉通知，那么需要填写两个变量，分别是 DD_BOT_TOKEN 和 DD_BOT_SECRET，两者不可缺一
