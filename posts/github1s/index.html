另一种方式在 GitHub 上阅读源码

发表于2021-03-19|更新于2021-03-19|实用分享
|字数总计:380|阅读时长:1分钟

前言

相信大家在 GitHub 上阅读源代码的时候，一个一个文件点啊点，结果还不是自己想要的，又得回退，很蛋疼，以至于去下载插件，比如 Octotree，装上之后，仓库的左侧就会有个树形的菜单，如图

但是只是多了个菜单，可能也无法满足某些人，u1s1，这插件确实挺好用的。

而本文讲的却是另一款工具，不是插件，也不是 .exe，而是 GitHub1s

项目地址：

What does 1s mean?

1s，大概就和项目描述所说的那样吧，1 秒钟阅读 GitHub 上的代码

how to use?

使用也挺简单

据项目的介绍说只需在 github 之后加个 1s，然后 Enter 即可阅读任何存储库

比如，我现在要阅读 Spring Boot 源码

地址是这个：https://github.com/spring-projects/spring-boot

打开之后很普通，没啥不同的

但是在 github 后加个 1s，你就会发现新大陆了

https://github1s.com/spring-projects/spring-boot

oh! 天哪，这不是我们熟悉的 VS Code 吗

有了这玩意，在 GitHub 上看源码感觉就和在本地一样，体验极其爽！但注意是只读的，你不能编辑它。

而且，据说这个 GitHub1s 在大陆也能直接秒开！

还有就是，上图中的底部说明也说的很清楚了，GitHub1s 不是 GitHub 推出的，只是一个开源项目，不要说啥 GitHub 官方推出了新工具 - GitHub1s，那样人家会笑你的
