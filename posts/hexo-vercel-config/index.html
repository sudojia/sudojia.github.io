Vercel托管你的静态博客

发表于2020-11-11|更新于2021-05-12|HexoVercel
|字数总计:432|阅读时长:1分钟

前言

本站已托管至 Vercel , 所以就想写一篇博客记录一下。 现已部署至 upyun + GitHub

注册登录

点击 Vercel 官网 并使用 GitHub 登录。

注意 GitHub 账户绑定的主邮箱📪，估计国内邮箱，比如 QQ 邮箱，163 邮箱会出现无法使用 GitHub 账号登陆 Vercel 的情况，推荐将 GitHub 主邮箱改为谷歌邮箱也就是 Gmail

Vercel 页面发生了小变化，前往 利用 GitHub + Vercel 来做视频床 查看，操作都是大同小异，主要是懒的编辑文章了

导入仓库

  • 点击 Overview 选择 Import Project

  • 然后选择导入Git 仓库

  • 复制自己的 GitHub 博客地址链接（这里指的是静态仓库，也就是你的 xxx.github.io 那个仓库，不是源码仓）

  • 等待导入成功

自定义域名

进入 dashboard，点击项目卡片 –> View Domains

如果你想使用Vercel 提供的二级域名，直接添加 xxx.now.sh 或 xxx.vercel.app 看你想要哪个，如果没有人用过的话，会自动验证成功。

如果你想绑定自己的域名，就在输入框中输入你想要绑定的网址，点击右侧 Add 即可。下方会提示 Invalid Config，然后在域名购买方控制台进行域名解析。解析完成后即可通过自己的域名访问自己的博客。

  • 如果是根域名，请在你的 DNS 解析处添加 A 记录，指向 76.76.21.21
  • 如果是子域名，请在你的 DNS 解析处添加 CNAME 记录，指向 cname.vercel-dns.com

至此 Vercel 的部署过程就结束了

如果你在部署过程中有什么问题，可以在下方留言。
